# Переименование файла с длинным именем

## Проблема
Файл имеет слишком длинное имя для Git на Linux (GitHub Actions):
`Графическая фигура Чашка с ручкой Оранжевым цветом выделен первоначальный бычи рывок красным цветом  основная фаза борьбы продавцов и покупателей.png`

## Решение

### 1. Переименовать файл

В Windows Explorer или через командную строку:

**Старое имя:**
```
public\патерны\Графическая фигура Чашка с ручкой Оранжевым цветом выделен первоначальный бычи рывок красным цветом  основная фаза борьбы продавцов и покупателей.png
```

**Новое имя:**
```
public\патерны\чашка_с_ручкой.png
```

### 2. Код уже обновлён

Ссылки в коде уже обновлены на новое имя файла.

### 3. Коммит и push

```powershell
cd "E:\7777\НОВАЯ АПКА\smart-trader-quest-main\smart-trader-quest-main"
git add .
git commit -m "Fix: Rename long filename to fix Git checkout error"
git push origin main
```

## Важно

После переименования файла в файловой системе, Git увидит это как удаление старого файла и добавление нового. Это нормально.
